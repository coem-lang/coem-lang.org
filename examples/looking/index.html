let me|I
let me be “looking”
print—I am “found”—

let se(e|a) be “blue”
print—see, sea—

let mis(t|sed) be “thick”
print—mist, missed—

let mou?rning be “dark”
print—morning is mourning—

let answer be nothing
    print—answer—
let answer be “this”
    print—answer—

    print—answer—

let you be “here”
let answer be you

Walkthrough

The first line sets up a variable using the let keyword. In regex, the pipe (vertical line) operator matches either the token on the left or the right, so this variable can be referred to using either me or I.

let me|I

It doesn’t have a value yet. On the next line, we assign it a value using the be keyword, so that it now holds the string value “looking”. 

let me be “looking”

We print the value on the next line. We ask if the variable I is equal to the string “found”, using the comparison keyword am. It isn’t, so false is printed in the output.

print—I am “found”—

In the next few blocks, different variables are created and compared.

This variable name matches both see and sea, meaning that when we later type see or sea, they’ll both give the same value.

let se(e|a) be “blue”

This variable name matches both mist and missed.

let mis(t|sed) be “thick”

This variable name uses a question mark, which matches either 0 or 1 of the previous token. Thus, the variable name matches either morning or mourning.

let mou?rning be “dark”

Newlines are important in the language—they separate different statements—but spaces and tabs are free for your poetic license, as the next block shows.

let answer be nothing
    print—answer—
let answer be “this”
    print—answer—
let answer be “that”
    print—answer—
let answer be 42
    print—answer—

We can also set variables to the value of other variables. Here, the variable you contains the string value “here”, so when we set the variable answer to the value of you, answer now also holds “here”.

let you be “here”
let answer be you
print—answer—
